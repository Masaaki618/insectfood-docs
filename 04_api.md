# API設計書 | 昆虫食初心者ガイド

---

## 1. 概要

| 項目      | 内容                                |
|---------|-----------------------------------|
| ベースURL  | `https://your-api.run.app/api/v1` |
| バージョン   | v1                                |
| 認証      | なし（MVP段階）                         |
| レスポンス形式 | JSON                              |

---

## 2. エンドポイント一覧

| メソッド | パス                    | 概要                |
|------|-----------------------|-------------------|
| GET  | `/api/v1/insects`     | 昆虫一覧取得            |
| GET  | `/api/v1/insects/:id` | 昆虫詳細取得            |
| GET  | `/api/v1/questions`   | 質問取得（カテゴリ別ランダム6問） |
| POST | `/api/v1/diagnosis`   | 診断実行（AIレコメンド）     |

}

## 4. AIプロンプト設計

### 診断結果用（POST /api/v1/diagnosis）

**渡すべき情報**

```json
{
  "scores": {
    "visual": 2,
    "physical": 1,
    "mental": 0
  },
  "insects": [
    {
      "id": 1,
      "name": "コオロギパウダー",
      "difficulty": 1,
      "taste": "ナッツ系",
      "texture": "サラサラ",
      "introduction": "粉末状なので虫とわからない初心者向け"
    }
  ]
}
```

**AIへの指示**

```
あなたは昆虫食のアドバイザーです。

以下のルールを必ず守ってください：
- 出力はJSON形式のみ。前後に説明文・マークダウン・コードブロックを一切つけない
- JSONのキー以外の文字列を出力しない
- ai_commentは必ず80文字以上100文字以内にする
- 必ず昆虫リストの中からinsect_idを1つ選ぶ
- ユーザーのスコアに基づいて最適な昆虫を選ぶ

【ユーザーの耐性スコア】
・見た目への耐性（visual）: {visual}/2点
・食べる勇気（physical）: {physical}/2点
・挑戦する気持ち（mental）: {mental}/2点

【選択可能な昆虫リスト】
{insects}

【良いコメントの例】
「挑戦する気持ちが強いあなたに、白くてまるっとした
 シルクワームはいかがでしょう。クリーミーな味わいに
 きっと驚くはずです。初めての昆虫食にぴったりです」

【悪いコメントの例（禁止）】
「おすすめです」「食べてみてください」などの薄い内容

出力形式（このJSONのみを返すこと）:
{"insect_id": 1, "ai_comment": "コメント"}
```

**AIから想定される情報**

```json
{
  "insect_id": 3,
  "ai_comment": "挑戦する気持ちが強いあなたに、白くてまるっとしたシルクワームはいかがでしょう。クリーミーな味わいにきっと驚くはずです。初めての昆虫食にぴったりです"
}
```

### 昆虫詳細用（GET /api/v1/insects/:id）

**渡すべき情報**

```json
{
  "name": "コオロギパウダー",
  "taste": "ナッツ系",
  "texture": "サラサラ",
  "introduction": "粉末状なので虫とわからない初心者向け"
}
```

**AIへの指示**

```
あなたは昆虫食のアドバイザーです。

以下のルールを必ず守ってください：
- 出力はJSON形式のみ。前後に説明文・マークダウン・コードブロックを一切つけない
- ai_commentは必ず80文字以上100文字以内にする
- 初心者が読んで食べてみたくなる内容にする
- 味・食感・見た目の特徴を具体的に盛り込む

【昆虫情報】
・名前: {name}
・味: {taste}
・食感: {texture}
・説明: {introduction}

【良いコメントの例】
「ナッツ系の香ばしい風味が特徴。粉末状なので虫の形が
 残らず初心者でも安心。スムージーやお菓子に混ぜるだけで
 手軽にタンパク質を補給できます」

【悪いコメントの例（禁止）】
「美味しいです」「食べてみてください」などの薄い内容

出力形式（このJSONのみを返すこと）:
{"ai_comment": "コメント"}
```

**AIから想定される情報**

```json
{
  "ai_comment": "ナッツ系の香ばしい風味が特徴。粉末状なので虫の形が残らず初心者でも安心。スムージーやお菓子に混ぜるだけで手軽にタンパク質を補給できます"
}
```

## 5. エラーレスポンス

| ステータスコード | 意味        | 例                         |
|----------|-----------|---------------------------|
| 400      | リクエスト不正   | スコアの値が不正                  |
| 500      | サーバーエラー   | DB接続エラー                   |
| 503      | AI APIエラー | Claude API呼び出し失敗（リトライ3回後） |

**エラーレスポンス形式**

```json
{
  "error": {
    "message": "invalid score value"
  }
}
```

**AI APIエラー時のデフォルトレスポンス**

```json
{
  "insect": {
    "id": 1,
    "name": "コオロギパウダー",
    ...
  },
  "ai_comment": "まずは食べやすいコオロギパウダーから始めてみましょう！"
}
```
